# Publish ACDesign Site Update Plan

**Goal:** Commit and publish the current site update, merge it into `develop`, then merge `develop` into `main`.

**Branch:** Start from the current branch, `feature/acdesignweb-foundation`, as requested.

## Checklist

- [x] Confirm `main` and `develop` branches exist.
- [x] Stop the local Jekyll preview server before branch/remote operations.
- [x] Verify GitHub CLI and remote repository access.
- [x] Run build and static checks before committing.
- [ ] Stage only the intended site update files.
- [ ] Commit the site update.
- [ ] Push `feature/acdesignweb-foundation`.
- [ ] Open a PR from `feature/acdesignweb-foundation` to `develop`.
- [ ] Merge the PR into `develop` and delete the remote feature branch.
- [ ] Open a PR from `develop` to `main`.
- [ ] Merge the PR into `main`.
- [ ] Provide a LinkedIn announcement message in English.
- [ ] Record final verification and PR results.
